# RAG / Workflow 架构说明｜听见鼓浪屿

## 推荐定位

本项目更适合先包装成 **Workflow + Context-aware RAG**，而不是直接说成完全自主 Agent。

原因是：当前系统最核心的价值是把输入、地图、用户类型、路线风险、遗产知识和验证指标组织成稳定流程。AI 不是自由行动，而是在被空间上下文约束后输出导览、问答和风险提醒。

## Workflow 节点

```text
1. 用户输入
   voice / text / photo / current location / destination / user type

2. 意图识别
   判断用户是在问路、问景点、问风险、问建筑、还是请求解释

3. 空间上下文组装
   读取当前位置、附近点位、路线节点、坡度、摩擦、风险等级和用户类型

4. 知识检索 / RAG
   从景点说明、遗产故事、路线注意事项和无障碍规则中检索相关片段

5. 路径风险计算
   基于距离、坡度、摩擦和用户 profile 生成路线权重

6. AI 回答生成
   输出导览说明、风险提示、路线解释或下一步行动建议

7. 验证与反馈
   用 Unity 模拟、hazard exposure、completion time 和用户反馈调整权重与 prompt
```

## RAG 知识库建议

可拆成四类知识：

| 知识库 | 内容 | 用途 |
|---|---|---|
| Heritage KB | 景点、建筑、历史故事、文化说明 | 生成导览和问答 |
| Accessibility KB | 坡度、台阶、湿滑、路面类型、避障规则 | 生成无障碍提醒 |
| Route KB | 节点、边、路线风险、转向说明 | 支持路线解释 |
| User Scenario KB | 老年、低视力、轮椅、普通游客场景 | 调整语气和推荐策略 |

## 为什么不是直接 Agent

当前不建议过度宣称为完全自主 Agent，因为：

- 用户安全相关，路线建议必须受规则和算法约束。
- AI 不应该自由规划未经验证的路线。
- 最有价值的部分是稳定 Workflow，而不是 Agent 自主决策。

如果后续升级为 Agent，可以增加：

- 工具调用：路线计算、景点检索、风险评分、语音纠错。
- 记忆：用户偏好、历史路线、避障偏好。
- 规划：根据目的地和用户状态选择服务流程。
- 评估：回答准确率、路线安全性、用户满意度。

## 评估指标

- 路线安全性：hazard exposure count。
- 导航效率：completion time / route distance。
- 无障碍适配：坡度、摩擦、台阶风险降低。
- AI 回答质量：是否引用正确地点、是否符合用户类型、是否避免幻觉。
- 用户体验：是否更容易理解路线和景点。

## 架构说明重点

我会把它定义为 Workflow-first 的 AI 空间导航产品。因为无障碍导航涉及安全和空间约束，所以我没有让大模型自由生成路线，而是让算法和知识库提供 grounding，再让 AI 做意图理解、解释和导览表达。后续可以加 Agent，但必须先保证工具、知识库和评估指标稳定。
