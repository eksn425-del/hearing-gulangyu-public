# Interview Talking Points

## 30-second Version

Hearing Gulangyu is a context-aware AI accessible guide for a heritage island. It combines AI API interaction, voice and photo questions, map-based site context, accessibility route algorithms, and Unity digital-twin validation. The key idea is that AI is not answering generically. It is grounded by Gulangyu's route network, heritage points, user profile, slope, friction, and risk context.

## 60-second Version

This project started as an accessibility-oriented navigation and heritage guide, but its product logic is closer to an AI spatial experience system. Visitors can interact through voice, text, or photos. The AI API helps interpret questions, correct speech recognition errors, and generate natural guide responses. I then constrain those responses with local map context, heritage points, route-risk algorithms, and user profiles such as elderly, wheelchair, or blind visitors. The routing engine calculates safer paths using distance, slope, friction, and user sensitivity. The Unity digital twin validates route behavior through digital cane sensing, hazard zones, exposure counts, and A/B comparison charts.

## How to Answer "Is This an AI Product?"

Yes, but I would describe it carefully. It is not just "an app with a chatbot." It is an AI-assisted spatial guide where the AI API is combined with map context, route algorithms, voice correction, photo interpretation, and digital-twin validation to improve the visitor experience.

## Avoid Overclaiming

- Do not say AI fully understands the site.
- Do not say the app guarantees safety.
- Do not say the AI replaces route algorithms.
- Do not present private API keys, raw user data, or deployment URLs.
- Do not claim a production-scale public deployment unless it is actually deployed.
