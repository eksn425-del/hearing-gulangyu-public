# 用户研究摘要｜听见鼓浪屿

## 目标用户

- 老年游客：需要更清晰、更慢速、更可靠的路线与导览说明。
- 低视力 / 视障游客：需要语音导览、风险提醒、方向提示和可感知路径信息。
- 轮椅或行动不便游客：需要避开高坡度、湿滑路面、台阶和拥挤巷道。
- 初次到访游客：需要在复杂历史街区中理解路线、景点和空间语境。
- 景区运营者 / 导览服务方：需要更可解释、更安全的导览辅助工具。

## 核心痛点

鼓浪屿这类历史街区的问题不是“找最短路”这么简单。真实游览过程中，用户会遇到：

- 巷道密集、岔路多，普通地图导航不够细。
- 坡度、路面摩擦、台阶、湿滑区域会影响无障碍体验。
- 文化遗产点位很多，游客需要结合当前位置理解讲解内容。
- 低视力或视障用户更依赖语音、方向、障碍和风险提示。
- 导览回答如果脱离地图和路线，就容易变成泛泛的旅游介绍。

## 为什么适合用 AI

AI 的价值不在于替代地图算法，而在于把多种上下文组织成可理解的服务输出：

```text
用户问题 + 当前位置 + 用户类型 + 目的地 + 地图节点 + 风险评分 + 遗产知识
-> 意图理解 / 语音纠错 / 图片问题解释
-> 结合路线风险和地点知识生成导览回答
-> 输出路线建议、风险提醒、转向说明或文化解释
```

## 仍需补充

- 真实访谈或问卷摘要。
- 3-5 个典型用户故事。
- 低视力 / 轮椅用户的任务场景截图或流程图。
- RAG 知识库的切块、召回和评估说明。

## 项目表达

这个项目解决的是历史街区中“地图导航不够感知场地风险和用户差异”的问题。我把它设计成一个 context-aware AI guide workflow：AI 负责理解用户问题和生成导览回答，路径算法负责风险约束，Unity 数字孪生负责验证路线行为。它不是泛聊天机器人，而是结合位置、用户类型、地图和遗产知识的空间智能产品。
