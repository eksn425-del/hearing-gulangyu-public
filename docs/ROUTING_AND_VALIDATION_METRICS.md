# Routing and Validation Metrics

## Route Algorithm Metrics

| Metric | Meaning |
| --- | --- |
| Distance | Route length in meters |
| Slope cost | Penalty for steep route segments |
| Surface friction | Penalty for slippery or difficult pavement |
| User profile weight | Different cost sensitivity for normal, elderly, wheelchair, or blind users |
| Difficulty score | Composite route burden indicator |
| Strategy | Safer route or shortest route |

## AI Interaction Metrics

| Metric | Meaning |
| --- | --- |
| Intent match | Whether the system understood the visitor's goal |
| Correction success | Whether noisy voice input was normalized correctly |
| Context relevance | Whether the AI response used map or heritage context |
| Safety mention accuracy | Whether accessibility warnings were added when needed |
| Clarification rate | How often the AI asks a follow-up question instead of guessing |

## Unity Validation Metrics

| Metric | Meaning |
| --- | --- |
| Route completion time | Time required for simulated agents to complete the route |
| Hazard exposure count | Number of contacts or exposures to risk zones |
| Hazard exposure ratio | Risk exposure normalized by route length or time |
| Digital cane detection | Whether ray-based sensing detects obstacles |
| A/B route comparison | Difference between baseline route and safer route |

## Product Interpretation

The strongest project claim is not that the app is perfect. The stronger claim is:

```text
I connected AI interaction with route algorithms and digital-twin validation, then used metrics to evaluate whether the experience became safer or more context-aware.
```
